2024 MidSemEasy Q2. COVID vs Multi-Disease System 4 M

A medical diagnostics company is building two AI systems:

  • Model A: Detects if a patient has COVID-19 or not (binary)
  • Model B: Classifies patients into one of five disease categories

(a) Justify shallow vs deep network for each model. (b) Which loss function suits each? Justify. (c) What is the role of Softmax in Model B? (d) Can Softmax be used in Model A?

(a) Model A — Shallow: Binary classification has a simpler decision boundary; a few layers with sigmoid suffice. Model B — Deep: 5-class, higher complexity, needs hierarchical features → more layers.
(b) Loss functions: Model A: Binary Cross-Entropy (BCE) — designed for y ∈ {0,1} problems. Model B: Categorical Cross-Entropy (CCE) — handles multi-class probability distribution.
(c) Role of Softmax in B: Converts raw logits (any real numbers) to a valid probability distribution where all values ∈ (0,1) and sum to 1. Enables argmax for final class prediction.
(d) Softmax in Model A: Not needed — with 2 classes, Softmax reduces to Sigmoid. Using Sigmoid (1 output neuron) is simpler and equivalent. Softmax would require 2 output neurons unnecessarily.

Module 2 — Perceptron: XOR Problem & Limitations

Jan 2026 MakeupMedium Q6. XOR Linear Separability 6 M

XOR function: (0,0)→0, (1,1)→0, (0,1)→1, (1,0)→1. Class 0 points: (0,0) and (1,1). Class 1 points: (0,1) and (1,0).

(i) Attempt to draw a straight line separating class 0 from class 1. Can you? Explain why or why not.

(ii) What fundamental limitation of the perceptron does this reveal?

(iii) Why does OR converge in 4 epochs but AND in 6 epochs, while XOR never converges?

(i) Cannot separate: Class 0 points are at (0,0) and (1,1) — the diagonal. Class 1 points are at (0,1) and (1,0) — the other diagonal. No single straight line can put all class-0 on one side and all class-1 on the other. The two classes alternate in a checkerboard pattern.
(ii) Limitation: A perceptron can only create a linear decision boundary (a hyperplane wTx = 0). Since XOR is not linearly separable, a single perceptron cannot solve it regardless of how many epochs it trains. It will always get stuck at 50% accuracy (two examples will always be misclassified).
(iii) OR vs AND convergence: OR has 3 positive examples and only 1 negative — the decision boundary can be found quickly because most updates push in the same direction. AND has only 1 positive and 3 negatives — it takes more updates to correctly isolate the single positive corner. XOR: no linear boundary exists → convergence is impossible (perceptron oscillates forever).

Solution to XOR: Multi-Layer Perceptron (MLP) with at least one hidden layer and non-linear activation can solve XOR.

Jan 2026 MakeupMedium Q9. Effect of Learning Rate 6 M

Gradient computed as ∇J = [5, 30, 35]T. Current weights w = [20, 8, 5]T.

(i) Perform weight update with η = 0.01. (ii) Perform update with η = 1.0. (iii) What potential problems does η = 1.0 cause? Explain with reference to convergence.

(i) η = 0.01: wnew = [20, 8, 5] − 0.01×[5, 30, 35] = [20−0.05, 8−0.3, 5−0.35] = [19.95, 7.7, 4.65]
(ii) η = 1.0: wnew = [20, 8, 5] − 1.0×[5, 30, 35] = [15, −22, −30]
(iii) Problems with large η: The step is too large — the update overshoots the minimum. Instead of moving downhill, the parameters jump to the other side of the error surface and climb up. This causes oscillation around the minimum and can even cause divergence (loss increases with each step). Rule of thumb: start with η = 0.001–0.01 and reduce if loss oscillates.

Dec 2025 MidSemMedium Q13. Threshold & Medical Diagnosis Case 6 M

A medical system detects disease. 1000 patients: 50 diseased, 950 healthy. Results: TP=40, FN=10, FP=95, TN=855.

(i) Calculate overall accuracy. Why is 89% accuracy misleading here? What would a naive "always predict healthy" classifier get?

Modules 5–6 — DFNN: Backpropagation

Mar 2026 EndSemHard Q19. Vanishing Gradients — Cause, Impact, Fixes 4 M

During DFNN training for a 10-class medical X-ray classifier, gradients near input layers vanish. (a) Identify the cause. (b) Explain the impact on learning. (c) Recommend TWO architectural fixes with justification.

(a) Cause: Using saturating activation functions (sigmoid or tanh) in hidden layers. Sigmoid derivative = σ(1−σ) has a maximum of 0.25. During backpropagation, gradients are multiplied by these derivatives at every layer via chain rule. For a 10-layer network: 0.25 × 0.25 × … → ≈ 10−6. Gradients become negligibly small before reaching early layers.
(b) Impact: Weight updates in early layers ≈ 0 → early layers stop learning meaningful features. The network effectively becomes shallow (only final layers train). For image classification, this means low-level features (edges, textures) are not learned, so the model fails to extract useful representations from raw pixels.
(c) Fix 1 — ReLU activation: f'(z) = 1 for z > 0. When gradient flows back through a ReLU neuron, it is multiplied by 1 (not <1). This preserves gradient magnitude across many layers, allowing deep networks to train properly.
Fix 2 — Residual connections (ResNet-style): y = F(x) + x. During backprop: ∂L/∂x = ∂L/∂y × (∂F/∂x + 1). The +1 identity term ensures the gradient never fully vanishes — it has a direct path back through the network without multiplicative shrinkage.

Modules 5–6 — DFNN: Architecture Design

Mar 2026 EndSemHard Q20. Churn Prediction — Complete DFNN Design 5 M

A telecom company predicts subscriber churn using 4 features: call duration, data usage, complaints, contract type. A logistic regression model achieves 88% accuracy but only 54% recall for churners.

Design a complete DFNN for this task. Specify: (i) input dimension, (ii) hidden layers with activation justification, (iii) output layer, (iv) loss function, (v) why DFNN is better than logistic regression here.

(i) Input: 4 neurons (one per feature; encode categorical "contract type" as 3 binary features → total 6 input neurons with one-hot encoding)
(ii) Hidden Layers:
Layer 1: 16–32 neurons, ReLU — cheap, no vanishing gradient, handles large dataset scale
Layer 2: 8–16 neurons, Leaky ReLU — deeper layer risks dying ReLU; Leaky ReLU (α=0.01 for negatives) keeps all neurons active and preserves churn pattern capacity
(iii) Output: 1 neuron, Sigmoid — outputs probability of churn P(churn|features) ∈ (0,1)
(iv) Loss: Binary Cross-Entropy (BCE) — paired with sigmoid; provides clean gradient (ŷ−y); appropriate for binary probability output
(v) DFNN vs Logistic Regression: Logistic regression only captures linear relationships. Hidden layers allow DFNN to learn non-linear interactions: e.g., "high data usage AND high complaints" is a non-linear churn signal. Layer 2 abstracts customer behaviour patterns (high-risk profile, low-risk loyal customer) that logistic regression misses with 54% recall.

Modules 6–8 — CNN: Architecture Questions

2024 EndSemMedium Q26. Why CNN for Images? ReLU vs Sigmoid? 2 M

(a) Explain in 1–2 sentences why a CNN is adequate for image classification.

(b) Why is ReLU preferred over sigmoid/tanh in modern CNNs? Provide two clear reasons.

(c) A CNN adds Batch Normalization after Conv1 and Conv3, and Dropout (p=0.5) before FC. Explain the role of each and their impact on training/generalization.

(a) CNN for images: CNNs exploit local receptive fields (spatially local features like edges) and weight sharing (same kernel applied across entire image), capturing spatial hierarchies and translation-invariant features efficiently — ideal for images where the same pattern can appear anywhere.
(b) ReLU advantages:
1. Non-saturating for z>0: derivative is exactly 1, so gradients flow unchanged through deep networks — mitigates vanishing gradient problem that plagues sigmoid/tanh.
2. Computationally cheap: max(0,z) requires no exponentials or divisions, much faster than sigmoid/tanh, and induces sparse activation (~50% neurons=0) for efficient representations.
(c) Batch Normalization: Normalizes layer activations to mean=0, variance=1 per batch (with learnable γ,β to restore expressiveness). Stabilises training, allows higher learning rates, reduces sensitivity to initialization, slightly regularises. Applied after Conv1/Conv3 to keep activation distributions stable across layers.
Dropout (p=0.5): Randomly zeros 50% of neurons during training. Prevents co-adaptation (neurons from relying on each other), acts as implicit ensemble of 2n subnetworks, reduces overfitting. Disabled at inference (multiply by 1−p to maintain expected output magnitude). Critical before FC which has the most parameters.

Mar 2026 EndSemHard Q29. Medical Classifier — Activation Functions, Loss, Metrics 5 M

A DFNN for skin cancer detection (7 classes) has: Hidden Layer 1 (ReLU), Hidden Layer 2 (Leaky ReLU), Output Layer (Softmax).

(a) Define the activation function for each layer with mathematical formulas.

(b) Explain why a perceptron cannot solve this 10-class task, referencing linear separability and decision boundaries.

(c) Compute the activation for Hidden Layer 1, neuron given: x=[0.8,−0.5,0.2], w=[0.4,0.1,−0.2], b=0.5. Compare ReLU vs Softmax for this neuron.

(a) Activation functions:
Hidden Layer 1 — ReLU: f(z) = max(0, z) = z if z > 0, else 0
Hidden Layer 2 — Leaky ReLU: f(z) = z if z > 0, else αz (α=0.01 typically)
Output Layer — Softmax: ŷk = ezk / Σj=17 ezj for k=1…7
(b) Perceptron limitation: A perceptron creates a single linear hyperplane boundary. For 7 lesion types, the feature space requires non-linear, multi-region boundaries — skin conditions share overlapping visual features (texture, colour) that no linear separator can correctly classify. The XOR analogy applies: just as XOR cannot be solved by a line in 2D, complex medical images cannot be separated by hyperplanes without hierarchical feature learning (hidden layers).
(c) Compute activation:
z = wTx + b = (0.4)(0.8) + (0.1)(−0.5) + (−0.2)(0.2) + 0.5 = 0.32 − 0.05 − 0.04 + 0.5 = 0.73
ReLU(0.73) = max(0, 0.73) = 0.73 (positive, passes through unchanged)
Softmax would be inappropriate for a single hidden neuron — it's designed for a vector of logits (all class scores) to produce a probability distribution. ReLU is correct for hidden layers: it introduces non-linearity cheaply, and the positive output 0.73 means this neuron is active and will contribute to the next layer's computation.
Dec 2025 MidSemMedium Q30. Mobile vs Cloud Deployment — Architecture Tradeoffs 3 M

Two architectures for mobile image classification (10 classes):

  • Architecture A: [256→128→64→10], 50K params, 5ms inference, 87% accuracy, 200MB memory
  • Architecture B: [256→512→512→256→128→10], 850K params, 45ms inference, 89% accuracy, 3.4GB memory

Evaluate for: (i) mobile deployment processing 1 million images/day. (ii) Would your recommendation change for cloud servers with unlimited resources?

(i) Mobile — Recommend Architecture A:

The 2% accuracy improvement (87% → 89%) does not justify 9× slower inference, 17× more parameters, and 3.4GB memory. Deploy Architecture A on mobile.
(ii) Cloud servers — Recommend Architecture B: With unlimited GPU compute, memory, and power, the bottleneck shifts to model quality. 2% accuracy × 1 million images/day = 20,000 additional correct predictions daily.
