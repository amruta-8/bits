"Which model should we deploy?" (Part-d evaluation)

Pick the metric that matches the domain Mobile / real-time / edge → favour the smaller, faster, lower-memory model (latency, RAM, battery beat a 1–2% accuracy gain). Numerical-setup hacks & common traps

Perceptron • Check the convention: \(z\ge0\) vs \(z>0\), and \(\{0,1\}\) vs \(\{-1,+1\}\).
• No update if the prediction is already correct — always compare \(\hat y\) to \(y\) first.
• Present one epoch as a table (z, ŷ, Δw, new w).
• Not linearly separable (XOR, "stuck at 50/75%") → needs a hidden layer.
Gradient descent • GD subtracts the gradient; the perceptron adds the correction — don't mix.
• Error vector is \(\hat y-y\) (predicted minus true).
• \(\eta\) too big → overshoot/diverge; too small → slow.
• Different feature scales → normalize for faster convergence.
Softmax / CCE • Subtract \(\max(z)\) before exponentiating (stability) — but the ratio is unchanged.
• Predicted class = argmax of the probabilities.
• CCE only counts the true class: \(-\log(\hat y_{\text{true}})\).
• Handy: \(e^{0.5}{=}1.65,\ e^{1}{=}2.72,\ e^{1.5}{=}4.48,\ e^{2}{=}7.39\).
Forward pass / params • Mind whether \(W\) is stored \([n_{out}\times n_{in}]\) or transposed — match the given vectors' shapes.
• Params per dense layer \(=(n_{in}\!\times\!n_{out})+n_{out}\); check whether biases are included.
• ReLU loses negatives (→0); sigmoid keeps them (→(0,0.5)).
• Overfitting risk if #params approaches #samples.
Metrics from a confusion matrix • TP = the cell; FP = column total − TP; FN = row total − TP.
• Precision \(=\frac{TP}{TP+FP}\) (of predicted-positives); Recall \(=\frac{TP}{TP+FN}\) (of actual-positives).
• Imbalanced data → quote the naive-classifier baseline; accuracy alone is misleading.
• Threshold ↓ → FP↑, FN↓ (recall↑).
Code blanks / bug-fixes • Memorise the 12 NumPy lines (see the code bank).
• Bug hot-spots: wrong sign (sigmoid \(1+e^{-z}\), GD \(-\), softmax negative), missing transpose (\(X^\top\)), wrong axis (softmax axis=1), argmin vs argmax, incomplete BCE.
• "What happens if uncorrected?": sign → diverges; transpose → shape crash; argmin → inverted predictions.

Reading "design / evaluate" questions

Two-step read (1) Identify the task from the wording → fixes the output activation, loss, and #output neurons (see the decision guide). (2) Identify the constraint (accuracy vs latency/memory/explainability) → fixes the architecture-size or model recommendation. State your choice, then justify each with one reason tied to the task/constraint. For parameter-count parts, write the per-layer formula and sum; for a constraint like "≤ 20,000 params", set up the inequality and solve for the hidden size.
MODULE 1

Introduction to Deep Learning

AI vs ML vs DL — the nesting dolls

These are three concentric circles, smallest inside largest:

AI (biggest) — the science of making machines do tasks that need human intelligence (includes rule-based systems with no learning, e.g. a chess engine of hand-written rules).
ML — a subset of AI where the machine learns from data/experience instead of being explicitly programmed.
DL (smallest) — a subset of ML that uses artificial neural networks with many layers to learn increasingly meaningful representations of data automatically.

The "deep" in deep learning refers to the number of layers (the model's depth) — successive layers each transform the data into a higher-level representation. Early layers learn simple features (edges), later layers learn complex concepts (faces).

Why deep learning now? (the 3 enablers) 1. Data — huge amounts of unstructured data (images, text, audio, video) + cheap storage + cheap sensors.
2. Compute — cheap GPUs and distributed clusters made training large nets feasible.
3. Algorithms — better learning methods, automatic feature generation (no hand-crafted features), scalability.

The 4 core components of any DL problem

This framework recurs in every module — examiners love asking you to identify these for a given scenario.

MODULE 2

The Perceptron

From biological to artificial neuron

Biological neuron: Dendrites receive signals → Soma (cell body) processes → Axon transmits output → Synapses connect to other neurons (knowledge stored in synaptic strengths).
Artificial neuron: inputs \(x_i\) each weighted by \(w_i\) (importance) → summed → activation \(f\) → one output \(\hat y\). Knowledge stored in weights.
Artificial neuron computation $$\hat y = f\!\left(w_0 + \sum_{i=1}^{d} w_i x_i\right) = f(\mathbf{w}^\top\mathbf{x})$$ \(w_0\) = bias (with input \(x_0=1\)) — lets the decision boundary shift away from the origin.

What is a Perceptron?

The perceptron (Rosenblatt, 1958) is the simplest artificial neuron: a weighted sum followed by a step (sign) activation. It's a linear binary classifier — its decision boundary \(\mathbf{w}^\top\mathbf{x}=0\) is a hyperplane.

Step activations (two conventions — read the question!) Binary \(\{0,1\}\): \(\hat y = 1\) if \(z\ge 0\), else \(0\).
Bipolar \(\{-1,+1\}\): \(\hat y = \text{sign}(z) = +1\) if \(z\ge 0\), else \(-1\).

Perceptron Learning Algorithm

The update rule (memorise) For each example, if misclassified, update every weight (including bias, with \(x_0=1\)): $$w_i \leftarrow w_i + \eta\,(y-\hat y)\,x_i$$ \(\eta\) = learning rate, \(y\) = true label, \(\hat y\) = predicted. If \(\hat y=y\) → no change.
  1. Initialise weights (often all zeros).
  2. For each training example: compute \(z=\mathbf{w}^\top\mathbf{x}\), then \(\hat y=\text{step}(z)\).
  3. If \(\hat y\neq y\): apply the update rule to all weights.
  4. Repeat over epochs until all examples are classified correctly (guaranteed only if data is linearly separable — Perceptron Convergence Theorem).
Exam shortcut "One epoch" = pass through all examples once, updating as you go. Present your work as a table: columns for \(x_1,x_2,t,z,y,\Delta w_0,\Delta w_1,\Delta w_2,\) new weights. Examiners explicitly ask for this tabular form.

Any linearly-separable Boolean function can be realised by one perceptron. Standard weight sets (with bias):

Why a single perceptron CANNOT learn XOR Plot the points: class-0 = (0,0) & (1,1) sit on one diagonal; class-1 = (0,1) & (1,0) sit on the other. No single straight line can separate them — they're not linearly separable. A perceptron only draws one hyperplane \(\mathbf{w}^\top\mathbf{x}=0\), so it gets stuck (≈50% accuracy forever). This is the fundamental limitation Minsky & Papert (1969) highlighted.
The fix → Multi-Layer Perceptron (MLP) Stack a hidden layer with non-linear activations. Each hidden neuron learns one linear separator; the output neuron combines them (e.g. XOR = "OR but NOT AND"). Hidden layers create new feature representations, enabling non-linear decision boundaries. This bridge motivates Module 5 (DFNN).

Linear separability

Two classes are linearly separable in \(n\)-D if a single \((n-1)\)-D hyperplane can split them (a line in 2-D, a plane in 3-D). If separable, infinitely many such lines exist and the perceptron will find one.

MODULE 3

Linear Neural Networks for Regression

What is regression?

A supervised task that predicts a continuous value \(y\in\mathbb{R}\) from features \(\mathbf{x}\in\mathbb{R}^d\) (e.g. house price, temperature, salary). Linear regression assumes the output is a linear combination of inputs.

Model = a single neuron with identity activation $$\hat y = w_0 + w_1x_1+\dots+w_dx_d = \mathbf{w}^\top\mathbf{x}$$ Vectorised over all \(N\) examples: \(\;\hat{\mathbf{y}} = X\mathbf{w}\), where \(X\in\mathbb{R}^{N\times(d+1)}\) (first column = 1s for bias).
Activation \(f(z)=z\) (identity): output can be any real number, derivative \(=1\) (gradient flows unchanged).
Objective — Mean Squared Error (MSE) $$J(\mathbf{w}) = \frac{1}{2N}\sum_{i=1}^{N}\big(\hat y^{(i)}-y^{(i)}\big)^2$$ Why squared error? Differentiable (smooth), convex (single global minimum for linear models), penalises large errors quadratically, and equals max-likelihood under Gaussian noise. The \(\tfrac{1}{2}\) just makes the gradient clean.

Some papers drop the \(\tfrac12\) and use \(J=\tfrac1N\sum(\hat y-y)^2\). Use whichever the question's code/marks imply; state your assumption.

Gradient Descent

Idea: iteratively step the weights downhill on the convex error surface (a bowl) until you reach the bottom (minimum).

Gradient & update (Batch GD) $$\nabla J = \frac{1}{N}\,X^\top(\hat{\mathbf{y}}-\mathbf{y}) \qquad\quad \mathbf{w} \leftarrow \mathbf{w}-\eta\,\nabla J$$ \(\eta\) = learning rate (step size). Batch GD uses all \(N\) examples each step.

One GD iteration — the exact recipe (Q2 in every paper)

  1. Predict: \(\hat y^{(i)} = \mathbf{w}^\top\mathbf{x}^{(i)}\) for each example.
  2. Loss: \(J=\tfrac{1}{2N}\sum(\hat y-y)^2\).
  3. Error vector: \(e=\hat{\mathbf{y}}-\mathbf{y}\).
  4. Gradient: \(\nabla J=\tfrac1N X^\top e\).
  5. Update: \(\mathbf{w}\leftarrow\mathbf{w}-\eta\,\nabla J\).

Learning rate \(\eta\) — the #1 conceptual trap

\(\eta\) too small\(\eta\) good\(\eta\) too large
Tiny steps → very slow convergence; may stall.Steady descent to the minimum.Overshooting — jumps over the minimum, oscillates or diverges (loss → ∞, NaN).
Feature scaling / normalization — why it speeds GD If features have very different ranges (e.g. area 400–2000 vs rooms 1–5), the error surface is a long thin valley → GD zig-zags and needs many epochs. Z-score normalization \(\big(\tfrac{x-\mu}{\sigma}\big)\) makes the surface more spherical/round → GD takes direct steps → converges in fewer epochs. (Final weights/MSE can be identical; only speed differs.)

Evaluation metrics (regression)

MSE \(=\tfrac1N\sum(\hat y-y)^2\) · RMSE \(=\sqrt{\text{MSE}}\) (same units as target) · MAE \(=\tfrac1N\sum|\hat y-y|\)
\(R^2 = 1-\dfrac{\sum(\hat y-y)^2}{\sum(\bar y-y)^2}\) — fraction of variance explained; 1 = perfect, 0 = no better than predicting the mean.

Watch units: if MSE = 5,000,000 then RMSE = √5,000,000 ≈ 2,236. Interpret RMSE relative to typical target values (e.g. a $2,236 error on $30k–$100k salaries ≈ 3–7% → acceptable).

Likely exam asks "Perform one batch-GD iteration: predict, MSE, gradient, update" · "Which learning rate gives larger change / what goes wrong if too large?" · "Which model converges faster — normalized vs raw features, and why (error-surface shape)?" · "Is this RMSE acceptable?".
MODULE 4

Linear Neural Networks for Classification

Classification predicts a discrete class. Three flavours: binary (\(y\in\{0,1\}\)), multi-class (one of \(K\) classes), multi-label (several classes at once).

Part A — Binary Classification (Logistic Regression)

Why not linear regression for classification? Linear output is unbounded \((-\infty,\infty)\) and not a probability; it's sensitive to outliers and gives a poor decision rule. We need an output squashed to \([0,1]\) → the sigmoid.
Sigmoid (logistic) function $$\sigma(z)=\frac{1}{1+e^{-z}}, \qquad \sigma'(z)=\sigma(z)\,(1-\sigma(z))$$ Range \((0,1)\) → read as \(P(y{=}1\mid x)\). Monotonic, smooth. \(\sigma(0)=0.5,\ \sigma'(0)=0.25\). Symmetry: \(\sigma(-z)=1-\sigma(z)\). As \(z\to+\infty\to1\), \(z\to-\infty\to0\).

Model: \(\hat y = \sigma(\mathbf{w}^\top\mathbf{x})\). Predict class 1 if \(\hat y\ge\) threshold (default 0.5, i.e. \(z\ge0\)).

Binary Cross-Entropy (BCE) loss $$\ell = -\big[\,y\log\hat y + (1-y)\log(1-\hat y)\,\big]$$ Derived from max-likelihood. Heavily penalises confident wrong predictions (loss → ∞). Convex for logistic regression; pairs perfectly with sigmoid.
The beautiful gradient (same form as regression!) For a single example: \(\;\nabla\ell = (\hat y - y)\,\mathbf{x}\). Over a batch: \(\;\nabla J = \tfrac1N X^\top(\hat{\mathbf y}-\mathbf y)\).
Update: \(\mathbf{w}\leftarrow\mathbf{w}-\eta\,\nabla\ell\). Trained with SGD (one example) or mini-batch SGD.

Part B — Multi-class Classification (Softmax)

One output neuron per class (K neurons). Labels are one-hot encoded: class \(k\) → vector of zeros with a 1 at position \(k\). Weight matrix \(W\in\mathbb{R}^{(d+1)\times K}\). Logits \(\mathbf{z}=W^\top\mathbf{x}\).

Softmax → probability distribution $$\hat y_k = \frac{e^{z_k}}{\sum_{j=1}^{K} e^{z_j}}$$ Outputs are in \((0,1)\) and sum to 1. Predicted class = \(\arg\max_k \hat y_k\). (Numerical-stability trick used in code: subtract \(\max(z)\) before exponentiating.)
Categorical Cross-Entropy (CCE) $$L = -\sum_{k=1}^{K} y_k \log \hat y_k$$ With one-hot \(y\), only the true class's term survives: \(L=-\log \hat y_{\text{true}}\). Gradient on logits: \(\nabla = \tfrac1B X^\top(\hat Y - Y)\) — same shape again. Trained with mini-batch SGD.

Softmax+CCE by hand (Q4 recipe)

  1. Exponentiate each logit: \(e^{z_0}, e^{z_1}, \dots\)
  2. Sum them.
  3. Divide each by the sum → probabilities.
  4. Predicted class = highest probability (argmax).
  5. CCE = \(-\log(\text{prob of TRUE class})\).

Handy values: \(e^{0.5}{\approx}1.649,\ e^{1.0}{\approx}2.718,\ e^{1.5}{\approx}4.482,\ e^{2.0}{\approx}7.389\).

Multi-class metrics

Per-class precision/recall from the confusion matrix: for class \(k\), TP = diagonal cell, FP = (column sum − TP), FN = (row sum − TP). Top-K accuracy: correct if the true class is among the K highest probabilities (used for 1000-class problems like ImageNet). Averaging: macro (mean over classes, treats all equally) vs micro/weighted (accounts for class sizes).

Deep Feed-Forward Neural Networks (DFNN / MLP)

Why we need depth & non-linearity

Linear models (one neuron) can only draw straight boundaries → fail on XOR & most real problems. Stacking layers with non-linear activations lets the network learn non-linear boundaries and hierarchical features.

Critical: without non-linearity, depth is useless A function is non-linear if \(f(ax_1+bx_2)\neq a f(x_1)+b f(x_2)\). If you stack linear layers only, the whole network collapses to a single linear layer (a product of matrices is just another matrix). Non-linear activations are what make deep networks powerful.

Architecture & forward pass

Layers: input → hidden(s) → output. For layer \(l\): \(\;\mathbf{z}^{(l)} = W^{(l)}\mathbf{h}^{(l-1)} + \mathbf{b}^{(l)}\), \(\;\mathbf{h}^{(l)} = \sigma^{(l)}(\mathbf{z}^{(l)})\). Output layer activation depends on task (identity/sigmoid/softmax).

Forward pass recipe (Q5 part a) 1. Hidden pre-activation \(\mathbf{z}^{[1]} = W^{[1]}\mathbf{x} + \mathbf{b}^{[1]}\)
2. Hidden activation \(\mathbf{a}^{[1]} = \text{ReLU}(\mathbf{z}^{[1]})\)
3. Output \(z^{[2]} = \mathbf{w}^{[2]}\mathbf{a}^{[1]} + b^{[2]}\), \(\;\hat y = \sigma(z^{[2]})\)
4. Loss (BCE/CCE/MSE per task)

⚠️ Mind whether \(W\) is stored as \([n_{out}\times n_{in}]\) or transposed — check shapes against the given vectors.

⭐ Parameter counting (asked in nearly every paper)

Fully-connected layer with bias $$\text{params} = (n_{in}\times n_{out}) + n_{out}$$ Total network = sum across all consecutive layer pairs. Example: [1000→8→5] = (1000·8+8)+(8·5+5) = 8008+45 = 8053.
MNIST [784→\(n_h\)→10]: \((784n_h+n_h)+(10n_h+10)=795n_h+10\). To keep ≤20000: \(n_h\le25\).
Overfitting rule of thumb Aim for #training-samples ≥ 5–10× #parameters. If params approach data size → high overfitting risk → reduce width/depth or regularize.

⭐ Backpropagation — the chain rule, backwards

Purpose: efficiently compute \(\partial J/\partial\) every weight by propagating an error signal \(\delta^{(l)}=\partial J/\partial \mathbf{z}^{(l)}\) from output back to input.

Key backprop equations (single example / batch of N) Output error: \(\;dZ^{[2]} = A^{[2]} - Y\) (for sigmoid+BCE / softmax+CCE)
Output weight grad: \(\;dW^{[2]} = \tfrac1N A^{[1]\top} dZ^{[2]}\)
Backprop to hidden: \(\;dA^{[1]} = dZ^{[2]} W^{[2]\top}\)
Through ReLU: \(\;dZ^{[1]} = dA^{[1]} \odot \mathbb{1}[Z^{[1]}>0]\)
Hidden weight grad: \(\;dW^{[1]} = \tfrac1N X^\top dZ^{[1]}\)
Update: \(\;W \leftarrow W - \eta\, dW\)

These exact lines are the most common code blanks / bug-fixes in Q5(b). Memorise the signs: it's always A - Y (predicted minus true) and weights subtract the gradient.

Universal Approximation Theorem (UAT)

UAT (Cybenko 1989; Hornik 1989): a feed-forward net with a single hidden layer of finitely many neurons can approximate any continuous function on a compact set (given a suitable activation). BUT it might need exponentially many neurons — deep networks achieve the same with far fewer parameters and learn better hierarchical representations.

Depth vs width & training challenges

  • Depth (more layers): hierarchical features (edges → shapes → objects), more expressive per parameter. Width (more units/layer): more capacity, higher overfitting risk.
  • Vanishing gradients: grads shrink in early layers (sigmoid/tanh) → no learning. Fix: ReLU, good init, batch norm.
  • Exploding gradients: grads blow up → NaN. Fix: gradient clipping, proper init.
  • Degradation: very deep nets harder to optimise. Fix: residual/skip connections (ResNet).

MODULE 6 · PARTIAL

Convolutional Neural Networks (CNN)

Scope note Your syllabus marks Module 6 as partial. There are no Session-6 slides yet, but CNN questions appear in past mid-sems (2024 papers) — almost always output-size and parameter-counting calculations plus a short "why CNN over ANN" concept. Focus there.

Why CNN over a plain ANN for images?

  • Local feature extraction: small filters slide over the image and learn local patterns (edges → textures → parts → objects) — a spatial hierarchy.
  • Weight sharing & sparse connections: the same filter is reused across the image → far fewer parameters than a fully-connected layer → less compute, less overfitting.
  • Preserves spatial information — a flattened ANN throws away the 2-D structure; CNN keeps it.

⭐ Output size formula (memorise)

Spatial size after conv / pool $$\text{Out} = \left\lfloor \frac{N - F + 2P}{S} \right\rfloor + 1$$ \(N\)=input size, \(F\)=filter size, \(P\)=padding, \(S\)=stride.
'valid' padding → \(P=0\). 'same' padding → output = input size (when \(S=1\)).
The depth of the output = number of filters \(K\) (output channels).

Example: input 8×8, filter 3×3, S=1, P=0 → (8−3)/1+1 = 6×6. With 5 filters → 6×6×5. Then 4×4 max-pool, S=2 → (6−4)/2+1 = 2×2, depth unchanged → 2×2×5.

⭐ Parameter counting in a CNN

Per layer Conv layer: \((F\times F\times C_{in} + 1)\times K\)  (filter weights + 1 bias per filter; \(C_{in}\)=input channels, \(K\)=#filters)
Pooling layer: 0 parameters (just downsampling).
Batch-norm layer: \(2\times C\) trainable (γ scale, β shift) + \(2\times C\) non-trainable (running mean, variance).
Flatten: 0 params (just reshape \(H\times W\times C\) → vector).
FC layer: \((n_{in}\times n_{out}) + n_{out}\).
Worked param example (RGB 64×64, Conv 64 filters 2×2) Weights = \(2{\times}2{\times}3{\times}64 = 768\), biases = 64 → 832 params. (Always include input channels \(C_{in}\)=3 for RGB!)

CNN building blocks & hyperparameters

  • Convolution layer — learnable filters detect features.
  • Pooling (max / average) — downsamples, adds translation invariance, controls overfitting (no params).
  • Fully-connected + output — final classification (softmax for multi-class).
  • Hyperparameters: #filters, kernel size, stride, padding, pooling size/stride, learning rate, batch size, epochs, dropout rate, weight init, optimizer, activation.
Transfer learning (sometimes asked) Train on a large dataset, then reuse/freeze early layers and fine-tune on a small dataset. Beats training from scratch on small data (which overfits) because early layers already learned general features.

P1. Perceptron one-step update — Dec 2025 Q1(a) · 6 marks
Question (verbatim) — Dec 2025 EC2 Regular, Q1(a) Consider a perceptron with 2 input features. Current weights are \(w_0=0.5\) (bias), \(w_1=0.3\) (weight for feature \(x_1\)), \(w_2=-0.4\) (weight for feature \(x_2\)). A training example has features \(x_1=1, x_2=2\) with true label \(y=1\). Learning rate \(\eta=0.1\). The step activation function outputs 1 if \(z\ge0\), otherwise outputs 0. Compute: (i) weighted sum \(z\), (ii) predicted output \(\hat y\) using step activation, (iii) if the example is misclassified, calculate the updated weights using the perceptron learning rule for each weight (note: treat bias \(w_0\) as having input \(x_0=1\)). [6 marks]
Solution

(i) Weighted sum. The perceptron first forms a linear combination of the inputs (the bias acts like a weight on a constant input \(x_0=1\)): $$z = w_0 + w_1x_1 + w_2x_2 = 0.5 + 0.3(1) + (-0.4)(2) = 0.5+0.3-0.8 = \mathbf{0.0}$$

(ii) Prediction. Apply the step function. Because the rule fires for \(z\ge0\) (note the "greater-than-or-equal"), and here \(z=0\), the boundary case still outputs 1: \(\hat y = \mathbf{1}\).

(iii) Weight update. The perceptron only updates weights when it gets an example wrong. Here \(\hat y = 1\) which equals the true label \(y=1\), so the example is correctly classified → no update is performed. The weights remain \([0.5,\,0.3,\,-0.4]\).

Had it been misclassified, you would apply \(w_i \leftarrow w_i + \eta(y-\hat y)x_i\) to every weight (including \(w_0\), with \(x_0=1\)). The term \((y-\hat y)\) is \(+1\) if we missed a positive, \(-1\) if we wrongly fired — it nudges the boundary toward the correct side.

Common trap: \(z\ge0\) (not \(z>0\)) means a point exactly on the boundary (\(z=0\)) is classified as 1. Many students lose the mark by assuming "no update needed" without first checking whether \(\hat y\) actually matches \(y\) — always state the comparison explicitly.

P4. Why XOR fails & how MLP fixes it — Jan 2026 Q1(a) · 6 marks
Question (verbatim) — Jan 2026 EC2 Makeup, Q1(a) For an XOR (exclusive OR) logical function, consider plotting the four points on a 2D coordinate plane where the x-axis represents \(x_1\) and the y-axis represents \(x_2\). The class 0 points (output = 0) are located at \((0,0)\) and \((1,1)\). The class 1 points (output = 1) are located at \((0,1)\) and \((1,0)\). (i) Attempt to draw a single straight line on this 2D plane that separates all class 0 points on one side from all class 1 points on the other side. Can you successfully draw such a line? Explain clearly why or why not, referring to the spatial arrangement of the four points. [3 marks] (ii) What fundamental limitation of the perceptron model does this reveal about its ability to learn certain types of functions? [3 marks]

(i) Can one straight line separate XOR? No. Plotting the four points, the class-0 points \((0,0)\) and \((1,1)\) sit at opposite corners of the unit square (one diagonal), while the class-1 points \((0,1)\) and \((1,0)\) sit at the other two opposite corners (the other diagonal). The two classes are interleaved diagonally. Any straight line that puts \((0,0)\) on one side and \((0,1),(1,0)\) on the other will always leave \((1,1)\) on the wrong side. Hence the data is not linearly separable — no single line works.

(ii) Fundamental limitation: a single perceptron computes only \(\hat y=\text{step}(w_0+w_1x_1+w_2x_2)\), whose decision boundary \(w_0+w_1x_1+w_2x_2=0\) is a single straight line (a hyperplane in higher dimensions). Therefore a perceptron can represent only linearly-separable functions. XOR requires a non-linear (or piecewise / two-line) boundary, which a single linear unit fundamentally cannot produce. This is exactly the limitation Minsky & Papert (1969) used to show single perceptrons are restricted, and it motivates multi-layer networks.

How an MLP fixes it: add one hidden layer with non-linear activations. Each hidden neuron draws its own linear separator, and the output neuron combines them (XOR = "OR" AND "NOT-AND"), giving a non-linear overall boundary. A minimal solution is the architecture 2 → 2 → 1: the hidden layer remaps the inputs into a new space where the two classes become linearly separable, and the output neuron then separates them.

P5. Different learning behaviour: AND vs OR vs XOR — Jan 2026 Q1(b) · 6 marks
Question (verbatim) — Jan 2026 EC2 Makeup, Q1(b) A student trains perceptrons to learn three different logical functions using identical initialization \((w=[0,0,0]^T)\) and learning rate \((\eta=0.1)\). The training results are: AND gate converges to 100% accuracy in 6 epochs; OR gate converges to 100% accuracy in 4 epochs; XOR gate after 100 epochs remains stuck at 50% accuracy (random guessing). Evaluate why these three functions exhibit such different learning behaviors: (i) Why XOR cannot reach 100% accuracy while AND and OR can? [3 marks] (ii) Why OR converges faster than AND (4 epochs versus 6 epochs), considering that OR has three positive examples while AND has only one. [3 marks]

(i) Why XOR never reaches 100% while AND/OR do. AND and OR are both linearly separable — a single straight line can perfectly split their positive and negative points. The Perceptron Convergence Theorem guarantees that, for linearly-separable data, the learning algorithm will find a separating boundary in a finite number of steps. XOR, however, is not linearly separable (its positives and negatives sit on opposite diagonals), so it violates the theorem's precondition. With no line able to satisfy all four examples, the weight updates from conflicting examples keep cancelling out — the perceptron oscillates and plateaus at ~50% (chance level) no matter how many epochs run.

(ii) Why OR converges faster than AND (4 vs 6 epochs). OR has three positive examples \(\{(0,1),(1,0),(1,1)\}\) and only one negative \((0,0)\); AND has only one positive \((1,1)\) and three negatives. Two effects make OR converge sooner: (1) Update frequency — more positive examples means more frequent corrective updates that push the weight vector in the right direction early on. (2) Easier boundary placement — OR only needs to fence off the single corner \((0,0)\) from everything else, a more "forgiving" boundary with a larger margin of valid lines, whereas AND must isolate the single corner \((1,1)\), so the algorithm needs more passes to position the line precisely.

P6. Minimal MLP to separate given 2-D points — Jul 2024 Q1–Q2 · 8+5 marks
Question (verbatim) — Jul 2024 EC2 Regular, Q1 & Q2 (Q1) \((x_1,x_2)\) are input features and target classes are either \(+1\) or \(-1\) as shown in the figure; \(x_2\) is never negative. How to attack any "minimum network for a decision region" question:

  • One line needed → single perceptron (no hidden layer).
  • Convex region bounded by \(k\) lines → 1 hidden layer with \(k\) nodes (one per line) + 1 output node doing AND of them.
  • Non-convex / disjoint region → 2 hidden layers (1st: the lines; 2nd: AND them into convex pieces; output: OR the pieces).

Per-line weights: for boundary \(ax_1+bx_2+c=0\), the neuron uses weights \([a,b]\) with bias \(c\), and you flip signs so the desired side gives a positive sum. Worked (triangle example): line through (3,0),(0,2): \(2x_1+3x_2-6=0\) → weights [2,3], b=−6. Line (3,0),(4,4): \(-4x_1+x_2+12>0\) → weights [−4,1], b=12. Line (0,2),(4,4): \(-x_1+2x_2-4=0\) → weights [−1,2], b=−4. Final AND neuron: weights [1,1,1], b=−3 (fires only when all three are 1).

XOR over 6 variables (Jun 2025 Q2c): single hidden layer needs \(2^{6-1}=32\) hidden perceptrons (one per odd-parity minterm); a deep network needs only \(O(n)\) — about \(3(6-1)=15\) neurons by chaining 2-input XORs. Depth is exponentially more parameter-efficient.

D8. Concept rapid-fire: Adam, dropout, bias, vanishing gradients, init — Jan 2025 Q2/Q6, Jun 2025 Q6

Adam β₁, β₂: β₁ (≈0.9) controls the exponential moving average of gradients (1st moment / momentum); β₂ (≈0.999) controls the EMA of squared gradients (2nd moment / adaptive scaling). Together → fast, adaptive convergence.

Dropout count: 6 nodes, each kept/dropped (2 choices) → \(2^6=\mathbf{64}\) possible thinned networks.

Bias term: lets the activation/decision boundary shift away from the origin, so the model can fit data not passing through 0 — added as \(w_0\) with input \(x_0=1\).

Vanishing gradients: in deep nets with sigmoid/tanh, gradients shrink layer-by-layer (derivatives <1 multiply) → early layers barely update → slow/stalled learning. Fix: ReLU (gradient 1 for z>0), good init (He/Xavier), batch norm, residual connections.

Weight init: too large → exploding gradients (unstable); too small → vanishing gradients (slow). All-zeros → symmetry (all neurons identical). Use He (ReLU) / Xavier (sigmoid-tanh).

Train 100% / test 45% (Jul 2024 Q3b): overfitting. Fix: L1/L2, dropout, early stopping, more/augmented data, simpler model, cross-validation.

"Train on 20 samples, high training loss; will 10,000 samples fix it?" (Jun 2025 Q6A): No — high training loss = underfitting (model too weak / bug / lr issue), not a data-quantity problem. E2. Why a Perceptron can't do 10 classes + vanishing gradients — Mar 2026 EC3 Regular, Q1(b)
Question (verbatim) (i) Explain why a Perceptron cannot solve this 10-class task, referencing linear separability, decision boundaries, and the XOR problem as analogy. [4 marks] (ii) During DFNN training, gradients near the input layers vanish. Identify the cause, explain the impact on learning, and recommend TWO architectural fixes with justification. [4 marks]

(i) Perceptron limitation. A perceptron forms only a single linear decision boundary (a hyperplane) per neuron, so it can only separate classes that are linearly separable. Real X-ray categories share overlapping visual features and need complex, non-linear decision regions that one hyperplane cannot carve out. Just as a single perceptron famously fails on XOR (whose classes can't be split by any straight line), it fails here. It also has no hidden layers, so it cannot learn intermediate feature representations (edges, textures, shapes) — a direct pixel-to-class linear map is far too weak for 10-way medical diagnosis. This is why we need a multi-layer, non-linear network.

(ii) Vanishing gradients. Cause: saturating activations (sigmoid/tanh) have derivatives \(<1\) (sigmoid's max is 0.25); backprop multiplies these derivatives across layers via the chain rule, so the product shrinks exponentially as it travels back to the early layers. Impact: gradients near the input become ≈0, so early-layer weights barely update — those layers stop learning useful low-level features and the network behaves as if it were shallow. Two architectural fixes: (1) ReLU-family activations — derivative is 1 for positive inputs, so gradients flow undiminished through active neurons. (2) Residual / skip connections — \(y=F(x)+x\) creates "gradient highways" that let gradients bypass layers and reach early layers strongly. (Batch-norm or He/Xavier init are also accepted.)

E3. Binary churn: activation, loss, threshold + logistic vs DFNN — Mar 2026 EC3 Regular, Q2(a)

(i) Activation + loss + threshold. Output activation = sigmoid \(\sigma(z)=\frac{1}{1+e^{-z}}\), giving a churn probability in \((0,1)\). (ii) Why a DFNN beats logistic regression here. The 88% accuracy with only 54% recall shows the linear model misses nearly half the churners. E4. Design a complete DFNN for churn + ReLU vs Leaky ReLU — Mar 2026 EC3 Regular, Q2(b)

E6. Overfitting diagnosis + weight-symmetry ratios — Mar 2026 EC3 Makeup, Q4
Question (verbatim) (A) A network shows training accuracy 99% and validation accuracy 72%. Diagnose the problem with evidence, and explain how Dropout helps. (B) Given weight symmetry where \(w_1=w_3\) and \(w_2=w_4\) (identical gradients), what are the ratios \(w_1/w_3\) and \(w_2/w_4\) throughout training and at convergence?

(A) Diagnosis: overfitting. Evidence: a large train-validation gap (99% vs 72%) with validation accuracy stagnating/falling while training keeps rising — the model is memorising training data, not generalising. Dropout fixes this by randomly zeroing a fraction of neurons each step (\(h_i=r_i a_i\) with \(r_i\sim\text{Bernoulli}\)): this prevents neurons from co-adapting (relying on specific partners), effectively trains an ensemble of sub-networks, and forces redundant, robust features — improving generalisation. (Early stopping, L2, more/augmented data also help.)

(B) Weight-symmetry ratios. If two weights start equal and always receive identical gradients, they update identically forever, so they stay equal: \(w_1/w_3=1\) and \(w_2/w_4=1\) at every step and at convergence. This is exactly why you must not initialise all weights equally (e.g. all zeros) — symmetric neurons never differentiate, and the layer learns only one distinct feature. Random initialisation breaks this symmetry.

Question (verbatim) (A) Training on 100×100 images for 5 classes. Net A = single linear layer + softmax. Net B = layers of size 128, 512, 32 + softmax (neither has hidden activations). With identical training, what do you expect about A vs B on test data? (B) After converging to a final loss, is it possible to reach a lower loss by ONLY changing the network initialization? (C) Four models' loss plots are shown — which is overfitting?

(A) Trick question: without hidden activations, stacking linear layers (128→512→32) still collapses to a single linear map, so Net B has the same representational power as Net A — a linear classifier. So you'd expect roughly similar (limited) test performance; B's extra parameters add no capacity (only redundancy / mild overfitting risk). The lesson: depth without non-linearity buys nothing. (If B did have ReLU between layers, it would have far more capacity and likely outperform A, at some overfitting risk.)

(B) Yes. Different random initialisations lead to different optimisation paths and different local minima/basins, so changing only the initialisation can land you at a lower final loss — initialisation genuinely matters.

(C) The overfitting model is the one whose training loss keeps decreasing while its validation loss rises/diverges (the two curves separate). High training accuracy with worsening validation = overfitting.

E8. Task type + end-to-end DL pipeline + computational graph — 2024 EC3 Regular, Q2
Question (verbatim) (A) An app must predict which kind of bird is in an image. Which kind of task is this? List and explain the steps you would follow to solve it using deep learning. (B) Draw the computational graph for \(o=a(w\cdot x+b)\) and for \(o=f(g_1(h_1(x),h_2(x)), g_2(h_2(x),h_3(x)))\).

E9. Reduce near-zero gradients + Dropout max/min (numeric) — 2024 EC3 Makeup, Q1
Question (verbatim) (A) Gradients close to zero can be a problem when training a network. Suggest four different techniques to reduce the problem. (B/C) For a dense layer with inputs/weights giving terms \(\{5{\cdot}2,\,1{\cdot}1,\,7{\cdot}1,\,2{\cdot}2\}\) and dropout \(p=0.5\) (inverted dropout scales surviving units by \(1/(1-p)=2\)), what are the maximum and minimum possible outputs? Give two interpretation cases each.

(A) Four fixes for vanishing (near-zero) gradients: (1) use ReLU-family activations (ReLU/Leaky ReLU) instead of sigmoid/tanh; (2) use proper weight initialization (He for ReLU, Xavier/Glorot for tanh); (3) add normalization layers (Batch/Layer Norm); (4) add residual / skip connections. (Gradient clipping helps the exploding case.)

(B) Maximum output. Inverted dropout divides surviving activations by \((1-p)=0.5\) (i.e. ×2). Case 1 — exactly 50% of nodes active: keep the two largest terms \(5{\cdot}2=10\) and \(7{\cdot}1=7\): \((10+0+7+0)/0.5=\mathbf{34}\). Case 2 — over many steps all nodes can be on: \((10+1+7+4)/0.5=\mathbf{44}\).

(C) Minimum output. Case 1 — 50% active, keep the two smallest: \((0+1{\cdot}1+0+2{\cdot}2)/0.5=(1+4)/0.5=\mathbf{10}\). Case 2 — all nodes dropped: output \(=\mathbf{0}\).

Key idea being tested: dropout randomly removes units during training and the survivors are rescaled by \(1/(1-p)\) so the expected activation is preserved.

E10. Init, multiclass loss, why backprop, batch-size vs iterations — 2024 EC3 Makeup, Q2
Question (verbatim) (A) After converging, is it possible to reach a lower loss by ONLY changing the network initialization? (B) What loss is commonly used for multi-class classification? (C) What is the key reason backpropagation is so important? (D) On a plot of "SGD iterations to reach a target loss vs batch size": (i) why do iterations decrease as batch size grows for small batches? (ii) why do iterations barely change for large batches?

(A) Yes — a different initialization can follow a different optimization path to a different (lower) minimum, even with everything else fixed.

(B) Categorical cross-entropy is standard for multi-class classification (paired with a softmax output).

(C) Backpropagation efficiently computes the gradient of the loss w.r.t. all parameters by applying the chain rule layer-by-layer, reusing intermediate results — this is what makes training large networks computationally feasible.

(D)(i) For small batches, a larger batch reduces gradient noise and gives a more accurate estimate of the true gradient, so each step is more effective → fewer iterations to hit the target. (ii) For large batches the gradient estimate is already accurate, so enlarging it further yields diminishing returns — convergence rate (in iterations) barely improves even though each iteration costs more compute.

E11. Represent a time-series dynamics exactly with one hidden layer (UAT/ReLU) — 2024 EC3 Makeup, Q6(I)
Question (verbatim) A time series satisfies \(u_{t+1}=5\max(2u_t-3u_{t-1}-u_{t-2}+1,\ 0)\) (with \(u_t=0\) for \(t<0\)). We want a neural network taking \(x=[u_{t-2},u_{t-1},u_t]^\top\) and returning \(y=u_{t+1}\). Show the dynamics can be represented exactly by a network with a single hidden layer — give the dimension, weights, and activation of both the hidden and output layers.

Solution. The recurrence is already in the form "linear combination → ReLU → scale", which is exactly one ReLU hidden unit followed by a linear output. So:

  • Hidden layer: dimension 1 (a single neuron), activation ReLU. Weights on \([u_{t-2},u_{t-1},u_t]\) are \(\mathbf{w=[-1,-3,2]}\) with bias \(\mathbf{b=+1}\). Its output is \(h=\text{ReLU}(2u_t-3u_{t-1}-u_{t-2}+1)=\max(2u_t-3u_{t-1}-u_{t-2}+1,0)\) — matching the inner term exactly.
  • Output layer: dimension 1, activation linear (identity), weight \(\mathbf{5}\), bias 0: \(y=5h=u_{t+1}\).

This is a concrete instance of the Universal Approximation Theorem idea: a single hidden layer with a non-linear activation (ReLU here) can represent this piecewise-linear dynamics exactly.
