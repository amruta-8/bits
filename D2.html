Session 1: Introduction to Deep Learning

1.1 What is Deep Learning?

Deep Learning is a subset of Machine Learning that uses neural networks with multiple layers (deep architectures) to learn hierarchical representations of data. Each layer transforms the input into progressively more abstract features.

The AI Hierarchy

Artificial Intelligence ⊃ Machine Learning ⊃ Representation Learning ⊃ Deep Learning

  • AI: Any technique enabling machines to mimic human behavior
  • ML: Algorithms that learn from data without explicit programming
  • Representation Learning: Automatically discovers features from raw data
  • Deep Learning: Learns multiple levels of representation via stacked layers
Key Insight: In traditional ML, feature engineering is manual. In deep learning, the network learns features automatically through its layers — raw pixels become edges, edges become textures, textures become object parts, parts become objects.

1.2 Why Deep Learning?

Data

Massive datasets now available (ImageNet, web-scale text). Deep learning performance scales with data — unlike traditional ML which plateaus.

Computation

GPUs, TPUs, and cloud computing make training deep networks feasible. What took weeks on CPUs takes hours on modern GPUs.

Algorithms

Key breakthroughs: ReLU activation (vanishing gradient fix), Dropout (regularization), Batch Normalization, better initialization (Xavier, He), advanced optimizers (Adam), and architectural innovations (ResNets, Transformers).

1.4 Learning Problems

Supervised Learning

Given input-output pairs $(x_i, y_i)$, learn a mapping $f: X \rightarrow Y$. Labels are provided.

Examples: Image classification, spam detection, regression

Unsupervised Learning

Given only inputs $\{x_i\}$, discover hidden structure or patterns. No labels.

Examples: Clustering, dimensionality reduction, generative models (GANs, VAEs)

Reinforcement Learning

An agent interacts with an environment, taking actions and receiving rewards. Goal: learn a policy that maximizes cumulative reward. No explicit input-output pairs.

Examples: Game playing (AlphaGo), robotics, autonomous driving

Session 2: Perceptron

2.1 Biological vs Artificial Neurons

Biological Neuron

  • Dendrites: Receive input signals
  • Soma (Cell body): Processes inputs
  • Axon: Transmits output signal
  • Synapse: Connection to next neuron (varying strengths)

Artificial Neuron

  • Inputs ($x_i$): Analogous to dendrites
  • Weights ($w_i$): Analogous to synaptic strengths
  • Summation ($\sum w_i x_i$): Analogous to soma
  • Activation function ($f$): Produces output

ANN Properties & Connectionism Principles

  • Massive parallelism: Many simple processing units operating simultaneously
  • Distributed representation: Knowledge stored across connection weights, not in individual units
  • Learning from experience: Weights adjusted based on training data
  • Fault tolerance: Graceful degradation — network works even if some units fail
  • Generalization: Ability to respond correctly to unseen inputs

2.2 The Perceptron Model

Perceptron (Rosenblatt, 1958): The simplest form of an artificial neuron. A single-layer binary classifier that computes a weighted sum of inputs and applies a step activation function.
$$\hat{y} = f\left(\sum_{i=0}^{n} w_i x_i\right) = f(w_0 + w_1 x_1 + w_2 x_2 + \cdots + w_n x_n)$$ where the activation function $f$ is the unit step function: $$f(z) = \begin{cases} 1 & \text{if } z \geq 0 \\ 0 & \text{if } z < 0 \end{cases}$$ $w_0$ is the bias (with $x_0 = 1$).
Geometric Interpretation: The perceptron defines a hyperplane $w_0 + w_1 x_1 + w_2 x_2 = 0$ that divides the input space into two half-spaces. Points on one side are classified as 1, the other as 0.

2.4 Perceptron Learning Algorithm

Algorithm (Iterative Weight Adjustment)

  1. Initialize weights $w_0, w_1, \ldots, w_n$ (e.g., to 0 or small random values)
  2. Set learning rate $\eta$ (e.g., $\eta = 1$)
  3. For each training example $(x, y)$:
    • Compute predicted output: $\hat{y} = f\left(\sum w_i x_i\right)$
    • Compute error: $e = y - \hat{y}$
    • Update weights: $w_i \leftarrow w_i + \eta \cdot e \cdot x_i$ for all $i$
  4. Repeat until all examples are correctly classified (or max iterations)
Perceptron Weight Update Rule: $$w_i^{new} = w_i^{old} + \eta \cdot (y - \hat{y}) \cdot x_i$$
  • If $y = \hat{y}$: no update (error = 0)
  • If $y = 1, \hat{y} = 0$: weights are increased toward the input
  • If $y = 0, \hat{y} = 1$: weights are decreased away from the input
Perceptron Convergence Theorem: If the training data is linearly separable, the Perceptron Learning Algorithm is guaranteed to converge to a solution in a finite number of steps. If the data is NOT linearly separable, the algorithm will never converge.

2.5 XOR Problem & Linear Separability

Linear Separability: A dataset is linearly separable if there exists a hyperplane that perfectly separates the two classes. A single perceptron can only solve linearly separable problems.
XOR Problem:
$x_1$$x_2$$y$ (XOR)
000
011
101
110

No single line can separate the 1s from the 0s. XOR is NOT linearly separable.

This was famously shown by Minsky & Papert (1969) and led to the first "AI Winter."

Solution: Use hidden layers (multi-layer perceptron). XOR can be decomposed as: $$\text{XOR}(x_1, x_2) = \text{AND}(\text{OR}(x_1, x_2),\; \text{NAND}(x_1, x_2))$$ This requires at least 2 layers — motivating the development of deep neural networks.
Session 3: Linear Neural Networks for Regression

3.1 Regression as Supervised Learning

Regression: A supervised learning task where the target variable $y$ is continuous (real-valued). The goal is to learn a function $f: \mathbb{R}^d \rightarrow \mathbb{R}$ that maps input features to a numeric output.

3.2 Linear Model as a Neuron

Linear Regression: $$\hat{y} = w_0 + w_1 x_1 + w_2 x_2 + \cdots + w_d x_d = \mathbf{w}^T \mathbf{x} + b$$ In vector form: $\hat{y} = \mathbf{X}\mathbf{w} + b$

Neural Network View

A linear regression model is equivalent to a single neuron with:

  • Inputs: $x_1, x_2, \ldots, x_d$
  • Weights: $w_1, w_2, \ldots, w_d$ (plus bias $w_0$)
  • Activation function: Identity $f(z) = z$
  • Output: $\hat{y} = f(\mathbf{w}^T\mathbf{x} + b) = \mathbf{w}^T\mathbf{x} + b$

3.3 Squared Error Loss (MSE)

Mean Squared Error: $$J(\mathbf{w}) = \frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2 = \frac{1}{n} \sum_{i=1}^{n} (y_i - \mathbf{w}^T \mathbf{x}_i)^2$$

Why MSE?

  • Differentiable: Smooth function allows gradient-based optimization
  • Convex: For linear models, MSE is a convex function of $\mathbf{w}$ — guaranteed global minimum
  • Gaussian noise interpretation: Minimizing MSE is equivalent to Maximum Likelihood Estimation (MLE) under the assumption that errors follow $\epsilon \sim \mathcal{N}(0, \sigma^2)$
  • Penalizes large errors: Squaring amplifies big deviations

3.4 Gradient Descent

Gradient Descent: An iterative optimization algorithm that updates parameters in the direction opposite to the gradient of the loss function, scaled by a learning rate $\eta$.
Update Rule: $$\theta_j := \theta_j - \eta \frac{\partial J}{\partial \theta_j}$$ For MSE with linear regression: $$\frac{\partial J}{\partial w_j} = -\frac{2}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i) x_{ij}$$

Batch Gradient Descent Algorithm

  1. Initialize parameters $\mathbf{w}$ randomly or to zeros
  2. Choose learning rate $\eta$ and number of iterations $T$
  3. Repeat for $t = 1, 2, \ldots, T$:
    • Compute predictions: $\hat{\mathbf{y}} = \mathbf{X}\mathbf{w}$
    • Compute gradients: $\nabla J = -\frac{2}{n}\mathbf{X}^T(\mathbf{y} - \hat{\mathbf{y}})$
    • Update: $\mathbf{w} \leftarrow \mathbf{w} - \eta \nabla J$
  4. Return $\mathbf{w}$
Learning Rate $\eta$ is critical:
  • Too large: Overshoots the minimum, may diverge
  • Too small: Converges extremely slowly
  • Just right: Steady convergence to minimum

Session 4: Linear Neural Networks for Classification

4.1 Binary Classification & Logistic Regression

Binary Classification: The target $y \in \{0, 1\}$. We need a model that outputs a probability $P(y=1|x) \in [0, 1]$.
Sigmoid Function: $$\sigma(z) = \frac{1}{1 + e^{-z}}$$

Sigmoid Properties

  • Range: $(0, 1)$ — outputs can be interpreted as probabilities
  • Monotonically increasing: Higher $z$ gives higher output
  • $\sigma(0) = 0.5$: Decision boundary at $z = 0$
  • Saturation: $\sigma(z) \to 1$ as $z \to \infty$; $\sigma(z) \to 0$ as $z \to -\infty$
  • Elegant derivative: $\sigma'(z) = \sigma(z)(1 - \sigma(z))$
Logistic Regression: $\hat{y} = \sigma(\mathbf{w}^T\mathbf{x} + b) = \frac{1}{1 + e^{-(\mathbf{w}^T\mathbf{x} + b)}}$

4.2 Binary Cross-Entropy Loss

Binary Cross-Entropy (Log Loss): $$\mathcal{L}(y, \hat{y}) = -\frac{1}{n}\sum_{i=1}^{n}\left[y_i \log(\hat{y}_i) + (1 - y_i)\log(1 - \hat{y}_i)\right]$$
Why not MSE for classification?
  • MSE with sigmoid creates a non-convex loss surface with many local minima
  • Cross-entropy is convex for logistic regression — unique global minimum
  • Cross-entropy gradient is stronger for confident wrong predictions, leading to faster learning

4.3 Stochastic Gradient Descent (SGD)

SGD: Instead of computing the gradient over the entire dataset (batch GD), SGD updates weights using one randomly selected example at each step.
SGD Update Rule: $$\mathbf{w} \leftarrow \mathbf{w} - \eta \nabla_{\mathbf{w}} \mathcal{L}(y_i, \hat{y}_i)$$ where $(x_i, y_i)$ is a single randomly chosen training example.

SGD Benefits

  • Fast: Each update is $O(d)$ instead of $O(nd)$
  • Memory efficient: Only needs one example in memory
  • Escapes sharp minima: Noise in gradient helps avoid bad local minima, often converging to flatter (more generalizable) solutions
  • Online learning: Can learn from streaming data

4.5 Multi-class Classification

One-Hot Encoding

For $K$ classes, represent label $y = k$ as a vector $\mathbf{y} \in \{0,1\}^K$ where $y_k = 1$ and all others are 0.

Example for 3 classes: class 1 = $[1,0,0]$, class 2 = $[0,1,0]$, class 3 = $[0,0,1]$

Softmax Function: $$\text{softmax}(z_k) = \frac{e^{z_k}}{\sum_{j=1}^{K} e^{z_j}} \quad \text{for } k = 1, 2, \ldots, K$$ Properties: All outputs $\in (0,1)$ and $\sum_k \text{softmax}(z_k) = 1$
Categorical Cross-Entropy Loss: $$\mathcal{L} = -\frac{1}{n}\sum_{i=1}^{n}\sum_{k=1}^{K} y_{ik} \log(\hat{y}_{ik})$$ Since $y_{ik}$ is one-hot, only the true class contributes: $\mathcal{L}_i = -\log(\hat{y}_{i,\text{true}})$

4.6 Mini-batch SGD

Mini-batch SGD

Compromise between Batch GD and SGD: compute gradient on a mini-batch of $B$ examples.

Session 5: Deep Feed-Forward Neural Networks

5.1 Why Linear Models Are Not Enough

Limitations of Linear Models:
  • Cannot learn XOR or any non-linearly separable function
  • Decision boundaries are always hyperplanes
  • Cannot capture feature interactions without manual feature engineering
  • Real-world data is rarely linearly separable

Solution: Stack multiple layers with non-linear activation functions to create Multi-Layer Perceptrons (MLPs).

5.2 MLP Architecture

Architecture: Input → Hidden → Output

  • Input Layer: Receives raw features. No computation — just passes values forward. Size = number of features $d$.
  • Hidden Layer(s): Apply linear transformation + non-linear activation. Each neuron: $h_j = \sigma(\mathbf{w}_j^T\mathbf{x} + b_j)$. These layers learn intermediate representations.
  • Output Layer: Produces final prediction. Activation depends on the task (identity for regression, sigmoid for binary, softmax for multi-class).

5.3 Activation Functions

PropertySigmoidTanhReLULeaky ReLU
Formula$\frac{1}{1+e^{-z}}$$\frac{e^z - e^{-z}}{e^z + e^{-z}}$$\max(0, z)$$\max(\alpha z, z)$
Range$(0, 1)$$(-1, 1)$$[0, \infty)$$(-\infty, \infty)$
Zero-centered?NoYesNoApprox.
Vanishing Gradient?Yes (severe)Yes (less severe)No (for $z>0$)No
Computational CostHigh ($e^x$)High ($e^x$)Very LowVery Low

Why ReLU works well:
  • No vanishing gradient for positive values — gradient is always 1
  • Computationally cheap: just a threshold at zero
  • Induces sparsity: some neurons output exactly 0
  • Biological plausibility: neurons either fire or don't
Dying ReLU Problem: If a neuron's input is always negative, it outputs 0 and gets 0 gradient. It "dies" and never recovers. This can happen with large learning rates or bad initialization.
Fix: Use Leaky ReLU ($\alpha = 0.01$), PReLU (learned $\alpha$), or ELU.

5.4 DFNN Definition & Layer Computation

Deep Feed-Forward Neural Network (DFNN): A neural network with at least two hidden layers. Information flows in one direction: input → hidden layers → output. No cycles or feedback connections.
Layer-wise Computation:

For layer $l$ ($l = 1, 2, \ldots, L$):

$$Z^{(l)} = H^{(l-1)} W^{(l)} + b^{(l)}$$ $$H^{(l)} = g^{(l)}(Z^{(l)})$$ where:
  • $H^{(0)} = X$ (input data)
  • $W^{(l)}$ = weight matrix of layer $l$ (shape: $n_{l-1} \times n_l$)
  • $b^{(l)}$ = bias vector of layer $l$ (shape: $1 \times n_l$)
  • $Z^{(l)}$ = pre-activation (linear combination)
  • $g^{(l)}$ = activation function of layer $l$
  • $H^{(l)}$ = post-activation (output of layer $l$)

5.5 Parameter Counting

Parameters per layer: $$\text{Params in layer } l = n_{l-1} \times n_l + n_l = n_l(n_{l-1} + 1)$$ where $n_{l-1}$ is the number of units in the previous layer and $n_l$ is the number in the current layer. Total parameters: $$\text{Total} = \sum_{l=1}^{L} n_l(n_{l-1} + 1)$$
Example: Input = 15 features, 4 hidden layers with 8 units each, 1 scalar output.
  • Layer 1: $15 \times 8 + 8 = 128$
  • Layer 2: $8 \times 8 + 8 = 72$
  • Layer 3: $8 \times 8 + 8 = 72$
  • Layer 4: $8 \times 8 + 8 = 72$
  • Output: $8 \times 1 + 1 = 9$
  • Total: $128 + 72 + 72 + 72 + 9 = 353$

5.6 Backpropagation

Backpropagation: An efficient algorithm for computing gradients of the loss with respect to all parameters in the network, using the chain rule of calculus applied layer by layer from output to input.

Backpropagation Algorithm

  1. Forward Pass: Compute $Z^{(l)}$ and $H^{(l)}$ for each layer $l = 1, \ldots, L$
  2. Compute Loss: $\mathcal{L} = \text{Loss}(H^{(L)}, Y)$
  3. Backward Pass: Compute error signal (delta) at output layer: $$\delta^{(L)} = \frac{\partial \mathcal{L}}{\partial Z^{(L)}}$$
  4. Propagate backwards: For $l = L-1, L-2, \ldots, 1$: $$\delta^{(l)} = \delta^{(l+1)} (W^{(l+1)})^T \odot g'^{(l)}(Z^{(l)})$$
  5. Compute parameter gradients: $$\frac{\partial \mathcal{L}}{\partial W^{(l)}} = (H^{(l-1)})^T \delta^{(l)}$$ $$\frac{\partial \mathcal{L}}{\partial b^{(l)}} = \sum \delta^{(l)}$$
  6. Update parameters: $$W^{(l)} \leftarrow W^{(l)} - \eta \frac{\partial \mathcal{L}}{\partial W^{(l)}}$$ $$b^{(l)} \leftarrow b^{(l)} - \eta \frac{\partial \mathcal{L}}{\partial b^{(l)}}$$
Key Result for BCE + Sigmoid output: $$\frac{\partial \mathcal{L}}{\partial Z^{(L)}} = \hat{Y} - Y = A - Y$$ This elegant result means the output error signal is simply the difference between predicted and actual values.

5.7 Depth vs Width & Universal Approximation

Width (More Neurons per Layer)

  • Can approximate any function with one hidden layer (UAT)
  • May need exponentially many neurons
  • Harder to train wide networks
  • Less parameter efficient

Depth (More Layers)

  • Learns hierarchical features
  • More parameter efficient
  • Each layer builds on previous abstractions
  • Risk of vanishing/exploding gradients
Universal Approximation Theorem (Cybenko, 1989): A feed-forward network with a single hidden layer containing a finite number of neurons can approximate any continuous function on compact subsets of $\mathbb{R}^n$, to arbitrary precision, given a suitable activation function (e.g., sigmoid).

Caveat: The theorem says such a network exists but doesn't say how to find it or how many neurons are needed. In practice, depth is more efficient than width.

5.8 Challenges & Design Considerations

Vanishing & Exploding Gradients

  • Vanishing: Gradients shrink exponentially as they propagate backward through many layers (especially with sigmoid/tanh). Early layers learn extremely slowly.
  • Exploding: Gradients grow exponentially, causing unstable updates and NaN values.
  • Solutions: ReLU activations, careful initialization (Xavier/He), Batch Normalization, skip connections (ResNets), gradient clipping.

Degradation Problem

As networks get deeper, training accuracy can actually decrease (not just test accuracy). This is NOT overfitting — it's a fundamental optimization difficulty. Solution: Residual connections (ResNets) where $H^{(l)} = F(H^{(l-1)}) + H^{(l-1)}$.

Weight Initialization

MethodFormula (Variance)Best With

CNN: Convolutional Neural Networks

Convolution Operation & Output Size

Convolution: A mathematical operation that slides a small kernel (filter) over the input, computing element-wise multiplication and summation at each position. It detects local patterns like edges, textures, and shapes.
Output Size Formula: $$O = \frac{W - F + 2P}{S} + 1$$ where:
  • $W$ = input size (width or height)
  • $F$ = filter/kernel size
  • $P$ = padding
  • $S$ = stride
Example: Input $8 \times 8$, kernel $3 \times 3$, stride $S=1$, padding $P=0$: $$O = \frac{8 - 3 + 0}{1} + 1 = 6$$ Output: $6 \times 6$. With 5 filters: output is $6 \times 6 \times 5$.

Pooling Layers

Pooling Operations

  • Max Pooling: Takes the maximum value in each pooling window. Most common. Retains strongest activations.
  • Average Pooling: Takes the average value. Smooths features. Often used before the final FC layer.

Pooling reduces spatial dimensions, reduces computation, and provides a degree of translation invariance. Uses the same output size formula as convolution (with $P=0$ typically).

Example: $6 \times 6 \times 5$ feature map, $4 \times 4$ max pool with stride 2 (non-overlapping $2 \times 2$ blocks don't apply here — $4 \times 4$ pool with stride 2): $$O = \frac{6 - 4}{2} + 1 = 2$$ Output: $2 \times 2 \times 5$

CNN Parameter Calculation

For a convolutional layer with $F \times F$ kernel, $C_{in}$ input channels, $C_{out}$ output filters:

$$\text{Parameters} = (F \times F \times C_{in} + 1) \times C_{out}$$

The $+1$ is for the bias per filter. Note: pooling layers have no learnable parameters.

Transfer Learning

Transfer Learning

Use a model pre-trained on a large dataset (e.g., ImageNet) and adapt it to a new task:

  • Feature extraction: Freeze convolutional layers, retrain only the FC classifier head
  • Fine-tuning: Unfreeze some/all layers and train with a low learning rate

Why it works: Early layers learn universal features (edges, textures) that transfer across tasks. Only task-specific high-level features need retraining.

RNN / GRU / LSTM

Vanilla RNN

Recurrent Neural Network (RNN): A neural network designed for sequential data where the hidden state at each time step depends on the current input and the previous hidden state, creating a form of memory.
Vanilla RNN Equations: $$h_t = \tanh(W_{hh} h_{t-1} + W_{xh} x_t + b_h)$$ $$y_t = W_{hy} h_t + b_y$$ where:
  • $h_t$ = hidden state at time $t$ (shape: $h$)
  • $x_t$ = input at time $t$ (shape: $d$)
  • $W_{xh}$ = input-to-hidden weights (shape: $d \times h$)
  • $W_{hh}$ = hidden-to-hidden weights (shape: $h \times h$)
  • $W_{hy}$ = hidden-to-output weights (shape: $h \times o$)
Vanilla RNN Parameter Count: $$\text{Parameters} = (d + h) \times h + h + h \times o + o$$ For the recurrent part only (no output layer): $(d+h) \times h + h$
Vanishing Gradient in RNNs: During backpropagation through time (BPTT), gradients are multiplied by $W_{hh}$ at each time step. If the largest eigenvalue of $W_{hh}$ is $< 1$, gradients vanish exponentially. If $> 1$, they explode. This makes learning long-range dependencies very difficult.

GRU (Gated Recurrent Unit)

GRU (Cho et al., 2014): A gating mechanism that controls information flow, allowing the network to learn when to update, reset, or preserve the hidden state. Simpler than LSTM (fewer parameters).
GRU Equations:
  • Update gate: $z_t = \sigma(W_z [h_{t-1}, x_t] + b_z)$ — how much of the old state to keep
  • Reset gate: $r_t = \sigma(W_r [h_{t-1}, x_t] + b_r)$ — how much of the old state to forget
  • Candidate hidden state: $\tilde{h}_t = \tanh(W_h [r_t \odot h_{t-1}, x_t] + b_h)$
  • New hidden state: $h_t = (1 - z_t) \odot h_{t-1} + z_t \odot \tilde{h}_t$
GRU Parameter Count: $$\text{Parameters} = 3 \times [(d + h) \times h + h]$$

The factor 3 comes from three sets of weights: update gate, reset gate, and candidate state. Each set has weights of shape $(d+h) \times h$ plus $h$ biases.

Example: $d = 10$ (input dim), $h = 20$ (hidden dim):
  • Vanilla RNN: $(10 + 20) \times 20 + 20 = 620$ parameters
  • GRU: $3 \times [(10 + 20) \times 20 + 20] = 3 \times 620 = 1860$ parameters
How GRU mitigates vanishing gradients: The update gate $z_t$ creates a direct additive path for the hidden state: $h_t = (1 - z_t) \odot h_{t-1} + z_t \odot \tilde{h}_t$. When $z_t \approx 0$, the hidden state is copied unchanged, allowing gradients to flow unimpeded across many time steps (similar to skip connections in ResNets).
Attention & Transformers

Scaled Dot-Product Attention

Attention Mechanism: Instead of compressing an entire sequence into a fixed-size vector, attention allows the model to focus on relevant parts of the input when producing each output element.
Scaled Dot-Product Attention: $$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$ where:
  • $Q$ = Query matrix (what am I looking for?)
  • $K$ = Key matrix (what do I contain?)
  • $V$ = Value matrix (what information do I provide?)
  • $d_k$ = dimension of keys (scaling factor to prevent large dot products from pushing softmax into saturation)

Step-by-Step Attention Computation

  1. Compute scores: $S = QK^T$ (dot product of query with all keys)
  2. Scale: $S' = S / \sqrt{d_k}$
  3. Normalize: $\alpha = \text{softmax}(S')$ (attention weights, sum to 1)
  4. Weighted sum: $\text{Output} = \alpha V$ (weighted combination of values)

Multi-Head Attention

Multi-Head Attention

Instead of one attention function, use $h$ parallel attention "heads," each with different learned projections:

$$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h) W^O$$ $$\text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V)$$

Benefits: Different heads can attend to different types of relationships (syntactic, semantic, positional) simultaneously.

Positional Encoding

Transformers have no inherent notion of order (unlike RNNs). Positional encodings are added to input embeddings:

$$PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d}}\right), \quad PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d}}\right)$$

These sinusoidal functions allow the model to learn relative positions and generalize to longer sequences.

Learning Rate Schedules

  • Step Decay: Reduce LR by factor every $k$ epochs. Simple and effective.
  • Exponential Decay: $\eta_t = \eta_0 \cdot e^{-\lambda t}$. Smooth decay.
  • Cosine Annealing: $\eta_t = \frac{\eta_0}{2}(1 + \cos(\pi t / T))$. Smooth cycle from $\eta_0$ to 0.
  • Warm-up: Start with very small LR, linearly increase for first few epochs, then decay. Important for Transformers.
  • ReduceOnPlateau: Reduce LR when validation loss stops improving. Adaptive and practical.
Convergence behavior:
  • SGD: Slow, zig-zag path, but often finds flatter minima (better generalization)
  • Momentum: Smoother trajectory, faster convergence in consistent gradient directions
  • Adam: Fast initial convergence, may settle in sharper minima (sometimes worse generalization)
  • In practice: Adam for quick experiments, SGD+Momentum for final tuning when generalization matters

Q3. Spam Classifier: Weights w=[0.2, 0.8, 0.9, -0.5]. Which feature most indicates spam? Why 75% accuracy plateaus?

Part A — Most indicative feature:

The weight with the largest positive magnitude has the strongest influence toward the positive (spam) class.

  • $w_1 = 0.2$ (e.g., number of links)
  • $w_2 = 0.8$ (e.g., presence of "free")
  • $w_3 = 0.9$ (e.g., number of exclamation marks)
  • $w_4 = -0.5$ (e.g., sender in contacts)

Feature 3 (weight = 0.9) most indicates spam because it has the highest positive weight.

Part B — Why 75% accuracy plateaus:

A perceptron is a linear classifier. If the data is not linearly separable, the perceptron cannot achieve better than a certain accuracy. The 75% plateau suggests approximately 25% of the data lies on the wrong side of any possible linear decision boundary. Solutions:

  • Use a multi-layer perceptron (MLP) to learn non-linear boundaries
  • Add more/better features to make the data more separable
  • Use a kernel method (SVM with non-linear kernel)

Q4. SGD: $t = (2p + 3)^2$, $p = 6$, $\eta = 0.1$, compute 3 iterations

Questions — Computational Graphs

Q5. Sigmoid Computational Graph: $f = \frac{1}{1 + e^{-z}}$, $z = 3$. Draw FP/BP, compute value and derivatives.

Computational Graph (decompose into primitive operations):

  1. $a = -z = -3$
  2. $b = e^a = e^{-3} = 0.0498$
  3. $c = 1 + b = 1.0498$
  4. $f = 1/c = 0.9526$

Forward Pass: $f = \sigma(3) = \frac{1}{1 + e^{-3}} \approx 0.9526$

Backward Pass (chain rule):

  1. $\frac{\partial f}{\partial c} = -1/c^2 = -1/(1.0498)^2 = -0.9070$
  2. $\frac{\partial c}{\partial b} = 1$
  3. $\frac{\partial b}{\partial a} = e^a = 0.0498$
  4. $\frac{\partial a}{\partial z} = -1$

$\frac{\partial f}{\partial z} = \frac{\partial f}{\partial c} \cdot \frac{\partial c}{\partial b} \cdot \frac{\partial b}{\partial a} \cdot \frac{\partial a}{\partial z}$

$= (-0.9070)(1)(0.0498)(-1) = 0.04517$

Verify with sigmoid derivative formula: $\sigma'(z) = \sigma(z)(1-\sigma(z)) = 0.9526 \times 0.0474 = 0.04517$ ✔

Answer: $f \approx 0.953$, $\frac{df}{dz} \approx 0.0452$

(Note: Some rounding gives 0.0548 depending on precision used. With $\sigma(3) = 0.9526$: $0.9526 \times 0.0474 = 0.0452$. The answer 0.0548 corresponds to a slightly different rounding.)

Q6. ReLU Computational Graph: $f = \text{relu}(ax + by + c)$, $a=3, b=2, c=-5, x=2, y=3$

Forward Pass:

  1. $u = ax = 3 \times 2 = 6$
  2. $v = by = 2 \times 3 = 6$
  3. $s = u + v + c = 6 + 6 + (-5) = 7$
  4. $f = \text{relu}(s) = \max(0, 7) = 7$

Backward Pass:

Since $s = 7 > 0$, $\frac{\partial f}{\partial s} = 1$ (ReLU derivative is 1 for positive inputs).

  • $\frac{\partial f}{\partial a} = \frac{\partial f}{\partial s} \cdot \frac{\partial s}{\partial u} \cdot \frac{\partial u}{\partial a} = 1 \cdot 1 \cdot x = 2$
  • $\frac{\partial f}{\partial b} = \frac{\partial f}{\partial s} \cdot \frac{\partial s}{\partial v} \cdot \frac{\partial v}{\partial b} = 1 \cdot 1 \cdot y = 3$
  • $\frac{\partial f}{\partial c} = \frac{\partial f}{\partial s} \cdot \frac{\partial s}{\partial c} = 1 \cdot 1 = 1$
  • $\frac{\partial f}{\partial x} = \frac{\partial f}{\partial s} \cdot \frac{\partial s}{\partial u} \cdot \frac{\partial u}{\partial x} = 1 \cdot 1 \cdot a = 3$
  • $\frac{\partial f}{\partial y} = \frac{\partial f}{\partial s} \cdot \frac{\partial s}{\partial v} \cdot \frac{\partial v}{\partial y} = 1 \cdot 1 \cdot b = 2$

Answer: $f = 7$, $\frac{\partial f}{\partial a} = 2$, $\frac{\partial f}{\partial b} = 3$, $\frac{\partial f}{\partial c} = 1$

Q7. XOR of 6 Parameters: How many perceptrons and layers needed?

Problem: Implement XOR of 6 boolean parameters: $y = x_1 \oplus x_2 \oplus x_3 \oplus x_4 \oplus x_5 \oplus x_6$

Approach: XOR of $n$ inputs can be decomposed into a tree of 2-input XOR operations. Each 2-input XOR requires 2 perceptrons (one hidden layer) plus 1 output perceptron.

Tree structure for 6 inputs:

  • Layer 1: XOR pairs — $(x_1 \oplus x_2)$, $(x_3 \oplus x_4)$, $(x_5 \oplus x_6)$ → 3 XOR units, each needing 2 hidden + 1 output = 9 perceptrons
  • Layer 2: XOR of first two results → 3 perceptrons
  • Layer 3: XOR of that result with third → 3 perceptrons

Answer: 15 perceptrons, 6 layers (counting all layers including intermediate hidden layers needed for each XOR sub-network in sequence).

Questions — Forward Pass & Backpropagation

Q10. Forward Pass: 2-layer DFNN with 2 inputs, 2 hidden (ReLU), 1 output (sigmoid). Compute all values and BCE loss.

Setup: Let inputs $x = [x_1, x_2]$, hidden layer weights $W^{(1)}$, biases $b^{(1)}$, output weights $W^{(2)}$, bias $b^{(2)}$.

General procedure:

  1. Hidden pre-activation: $z^{(1)} = W^{(1)T}x + b^{(1)}$ (vector of 2 values)
  2. Hidden activation (ReLU): $h^{(1)} = \max(0, z^{(1)})$
  3. Output pre-activation: $z^{(2)} = W^{(2)T}h^{(1)} + b^{(2)}$ (scalar)
  4. Output activation (Sigmoid): $\hat{y} = \sigma(z^{(2)}) = \frac{1}{1 + e^{-z^{(2)}}}$
  5. BCE Loss: $\mathcal{L} = -[y\log(\hat{y}) + (1-y)\log(1-\hat{y})]$

Key points:

  • ReLU zeros out negative pre-activations
  • Sigmoid squashes output to $(0, 1)$
  • BCE loss approaches 0 when $\hat{y}$ matches $y$, and $\infty$ when they maximally disagree

Substitute given weights/biases and compute step by step.

Q11. Backprop: Derive $\frac{\partial \mathcal{L}}{\partial Z}$ for BCE loss with sigmoid output

Given:

  • $\hat{y} = A = \sigma(Z)$
  • $\mathcal{L} = -[Y\log(A) + (1-Y)\log(1-A)]$

Derivation:

$\frac{\partial \mathcal{L}}{\partial A} = -\frac{Y}{A} + \frac{1-Y}{1-A} = \frac{A - Y}{A(1-A)}$

$\frac{\partial A}{\partial Z} = \sigma(Z)(1-\sigma(Z)) = A(1-A)$

By the chain rule:

$$\frac{\partial \mathcal{L}}{\partial Z} = \frac{\partial \mathcal{L}}{\partial A} \cdot \frac{\partial A}{\partial Z} = \frac{A - Y}{A(1-A)} \cdot A(1-A) = A - Y$$

Answer: $\frac{\partial \mathcal{L}}{\partial Z} = A - Y$

This elegant result is one of the most important in deep learning. The gradient is simply the difference between predicted and actual values, making computation very clean.

Questions — Overfitting & Diagnostics

Q16. Overfitting: 100% train accuracy, 45% test accuracy. Inferences and solutions.

Inference:

  • The model has severely overfit to the training data
  • It has memorized training examples rather than learning generalizable patterns
  • The large gap (100% - 45% = 55%) indicates very high variance
  • The model is too complex for the amount of training data

Solutions:

  1. More training data: Most effective solution if available
  2. Regularization: L2 (weight decay), L1 regularization
  3. Dropout: Randomly drop neurons during training (typical: p=0.5 for hidden, p=0.2 for input)
  4. Early stopping: Monitor validation loss, stop when it starts increasing
  5. Data augmentation: Artificially expand training set (flips, rotations, crops for images)
  6. Reduce model complexity: Fewer layers, fewer units per layer
  7. Batch normalization: Can have mild regularizing effect
  8. Cross-validation: Use k-fold CV for better model selection
Q17. Confusion Matrix: 3-class animal classifier. Compute precision/recall for Bird class.

Assume the following confusion matrix (3 classes: Cat, Dog, Bird):

  • True Positives (TP): Actually Bird, predicted Bird = 45

  • Recall = (diagonal element) / (row sum)

GRU:

$$\text{Params} = 3 \times [(d + h) \times h + h] = 3 \times [(10 + 20) \times 20 + 20] = 3 \times 620 = 1860$$

Breakdown: 3 gates (update $z$, reset $r$, candidate $\tilde{h}$), each with $(d+h) \times h$ weights + $h$ biases.

Answer: Vanilla RNN = 620, GRU = 1860

GRU has exactly 3x the parameters of a vanilla RNN (LSTM has 4x due to 4 gates: input, forget, output, cell candidate).

Q19. Scaled Dot-Product Attention: Product recommendation with $q_3 = [2, 7]$, given keys and values

Setup: Query $q_3 = [2, 7]$ (user preference vector), $d_k = 2$.

Assume keys and values for 3 products:

ProductKey $k_i$Value $v_i$
1$[1, 0]$$[10, 5]$
2$[0, 1]$$[3, 8]$
3$[1, 1]$$[7, 7]$

Step 1: Compute scores $s_i = q \cdot k_i$

  • $s_1 = [2,7] \cdot [1,0] = 2$
  • $s_2 = [2,7] \cdot [0,1] = 7$
  • $s_3 = [2,7] \cdot [1,1] = 9$

Step 2: Scale by $\sqrt{d_k} = \sqrt{2} \approx 1.414$

  • $s_1' = 2/1.414 = 1.414$
  • $s_2' = 7/1.414 = 4.950$
  • $s_3' = 9/1.414 = 6.364$

Step 3: Softmax

  • $e^{1.414} = 4.113$, $e^{4.950} = 141.18$, $e^{6.364} = 580.05$
  • Sum $= 725.34$
  • $\alpha = [0.0057, 0.1946, 0.7997]$

Step 4: Weighted sum of values

$$\text{Output} = 0.0057 \times [10,5] + 0.1946 \times [3,8] + 0.7997 \times [7,7]$$ $$= [0.057, 0.029] + [0.584, 1.557] + [5.598, 5.598] = [6.24, 7.18]$$

Interpretation: The model attends most strongly to Product 3 (attention weight 0.80), moderately to Product 2 (0.19), and barely to Product 1 (0.006). The output recommendation vector is closest to Product 3's value.
