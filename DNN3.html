📈 Session 3 — Linear Regression
Model: ŷ = wTx = w0 + w1x1 + … (identity activation)
Vectorized: Ŷ = Xw (X is N×(d+1) design matrix)
MSE Loss: J = (1/N)‖Xw − y‖²
Gradient: ∇J = (2/N) × XT(Xw − y) = (2/N)XTe
Update: w ← w − η × ∇J

Why MSE: Differentiable, convex (single global minimum for linear model), max-likelihood under Gaussian noise

Feature scaling: z = (x−μ)/σ → spherical error surface → faster convergence

Learning rate η: Too small → slow. Too large → diverges/oscillates. Use η ∈ {0.1, 0.01, 0.001}.

Adam default: η=0.001, β₁=0.9, β₂=0.999 — adaptive per-parameter learning rates

🔵 Session 4 — Binary Classification (Sigmoid + BCE)
Sigmoid: σ(z) = 1/(1 + e−z)  Range: (0,1)
σ'(z): σ(z)(1 − σ(z))  Max = 0.25 at z=0
BCE Loss: L = −[y log(ŷ) + (1−y) log(1−ŷ)]
Key gradient: ∂L/∂z = ŷ − y (just error!)
Weight grad: ∇wL = (ŷ − y) × x
Decision: Predict 1 if ŷ ≥ 0.5 (adjustable threshold)

Why sigmoid+BCE? Chain rule gives ∂L/∂z = ŷ−y cleanly — stable gradients, no saturation issues in gradient computation.

Threshold 0.5: Balanced FP/FN. Lower threshold → more positives → higher recall, lower precision.

🟣 Session 4 — Multi-class (Softmax + CCE)
Logits: zk = wkTx (one per class)
Softmax: ŷk = ezk / Σj ezj
Properties: Σk ŷk = 1 | ŷk ∈ (0,1)
CCE Loss: L = −Σk yk log(ŷk) (one-hot y)
Prediction: argmaxk(ŷk)
One-hot eg: Class 2 of 4 → y = [0,1,0,0]T

Numerical stability: subtract max(z) before computing ezk to avoid overflow

Softmax with 2 classes = equivalent to Sigmoid logistic regression

📊 Evaluation Metrics
Accuracy: (TP+TN)/(TP+TN+FP+FN)
Precision: TP/(TP+FP) ← of predicted +, how many right?
Recall: TP/(TP+FN) ← of actual +, how many found?
F1: 2×P×R/(P+R)
When to UseMetric
Balanced classesAccuracy
FP costly (spam)Precision
FN costly (disease)Recall
Imbalanced datasetF1 Score

Accuracy paradox: 99k legit + 1k fraud → always predict legit = 99% accuracy, 0% recall. ALWAYS check recall for imbalanced problems!

🧠 Sessions 5–6 — DFNN Architecture
Forward pass:
Z[l] = H[l−1] W[l] + b[l]
H[l] = f[l](Z[l])  [ReLU for hidden]
Backprop:
δ[L] = ŷ − y (output error)
δ[l] = (δ[l+1] W[l+1]T) ⊙ f'(Z[l])
∂J/∂W[l] = H[l−1]Tδ[l]/N
W[l] ← W[l] − η × ∂J/∂W[l]

Params per layer: (nin × nout) + nout

Example: 784→200→100→10 = 157,000+20,100+1,010 = 178,110

⚡ Activation Functions
FunctionFormulaRangeUse
Sigmoid1/(1+e−z)(0,1)Binary output ONLY
Tanh2σ(2z)−1(−1,1)Small nets, zero-centred
ReLU ⭐max(0,z)[0,∞)Hidden layers (default)
Leaky ReLUmax(αz,z) α=0.01(−∞,∞)Deep nets, fixes dying ReLU
Softmaxezk/Σezj(0,1)Multi-class output ONLY

Why ReLU: f'(z)=1 for z>0 → no vanishing gradient, cheap to compute, sparse

Dying ReLU: neuron always outputs 0 → gradient=0 → stuck. Fix: Leaky ReLU

Sigmoid issue: σ'(z) max=0.25 → gradients multiply by <0.25 at every layer → vanish in deep nets

⚠️ Vanishing Gradients — Cause & Fixes

Cause: Sigmoid/Tanh derivatives are small (<1). Backprop multiplies these at every layer → 0.25 × 0.25 × … → 0 in early layers

Effect: Early layers stop learning; only final layers train; network behaves like a shallow model

FixMechanism
ReLUf'(z)=1 for z>0 → gradient flows unchanged
Residual connectionsy=F(x)+x → ∂L/∂x gets +1 term, never fully vanishes
Batch NormNormalises activations → stable gradient magnitudes
He InitReLU: w ~ N(0, 2/nin); Xavier: N(0, 1/nin) for sigmoid/tanh
🖼️ Sessions 6–8 — CNN Core Formulas
Conv output size: ⌊(n + 2P − K) / S⌋ + 1
n=input, P=padding, K=kernel, S=stride
Same padding: P=(K−1)/2 → output = input (S=1)
Conv params: K × K × Cin × Cout + Cout
Pool output: ⌊(n − k) / s⌋ + 1 (P=0 for standard pooling)
Pool params: 0 (no learnable parameters!)
Flatten: H × W × C → single vector
FC params: (H×W×C) × nout + nout

3 properties exploited: Locality (local receptive field), Translation invariance (weight sharing), Hierarchy (edges→shapes→objects)

🏗️ Classic CNN Architectures
ModelInnovationKey Detail
LeNetFirst CNN (1998)Conv+Sigmoid+Pool; MNIST
AlexNetGPU training (2012)ReLU, Dropout, Data aug; ImageNet winner
VGGNetOnly 3×3 kernelsVery deep; 16–19 layers
InceptionParallel filters1×1, 3×3, 5×5 in parallel; 1×1 bottleneck
ResNetSkip connectionsy=F(x)+x; 50–152+ layers; solved degradation

ResNet key equation: y = F(x) + x  →  ∂L/∂x = ∂L/∂y × (∂F/∂x + 1). The +1 prevents gradient vanishing.

1×1 conv: Changes channel count only (Cin→Cout); adds non-linearity; reduces parameters before 3×3 conv.

📐 Depth vs Width & Design Guidelines
DeeperWider
FeaturesHierarchicalParallel
Param efficiencyBetterLower
RiskVanishing gradOverfitting

Universal Approx Theorem: Single hidden layer with finite neurons can approximate any continuous function — but deep nets are more efficient.

Start with: 2–3 hidden layers, 64–256 neurons, ReLU, then scale up if needed.

Decreasing width pattern: 1000 → 512 → 256 → 10 (funnels into class probabilities)

🔧 Overfitting Diagnosis & Fixes
ConditionTrain LossVal LossDiagnosis
UnderfittingHighHighModel too simple or too few epochs
OverfittingLowHighMemorising training data
Well-regularisedLowLowGood generalisation

Fixes for overfitting:

  • Dropout: randomly zero neurons during training (p=0.5); ensemble effect
  • L2 Regularization: add λ‖w‖² to loss → weight decay
  • Early Stopping: stop when val loss starts rising
  • Data Augmentation: flip, crop, rotate (for images)
  • Batch Norm: normalises activations → faster training, slight regularisation

💻 Python Code Patterns (Code Completion Questions)

Perceptron

z = np.dot(X[i], weights)
y_pred = 1 if z >= 0 else 0
error = y[i] - y_pred
weights += lr * error * X[i]

Logistic Regression

def sigmoid(z):
  return 1/(1+np.exp(-z))
z = np.dot(X_batch, weights)
y_pred = sigmoid(z)
gradient = X_batch.T @ (y_pred-y)
weights -= lr * gradient / N

Softmax

def softmax(Z):
  e = np.exp(Z-Z.max(1,keepdims=True))
  return e / e.sum(1, keepdims=True)
Z = X_batch @ W
Y_pred = softmax(Z)
gradient = X_batch.T @ (Y_pred-Y_batch)
W -= lr * gradient / N

DFNN Forward

Z1 = X @ W1 + b1
A1 = np.maximum(0, Z1) # ReLU
Z2 = A1 @ W2 + b2
A2 = sigmoid(Z2) # output

DFNN Backward

dZ2 = A2 - Y # BCE+sigmoid
dW2 = A1.T @ dZ2 / N
dA1 = dZ2 @ W2.T
dZ1 = dA1 * (Z1 > 0) # ReLU deriv
dW1 = X.T @ dZ1 / N

Linear Regression

y_pred = X @ weights
loss = np.mean((y_pred-y)**2)
error = y_pred - y
gradient = (2/N) * X.T @ error
weights -= lr * gradient

Common errors in code questions: wrong sign in gradient (should be y_pred−y, not y−y_pred), missing division by N, wrong axis in softmax (axis=1 not axis=0), wrong activation (argmin instead of argmax for prediction)

✅ Architecture Design Checklist
  1. Input dimension: State it explicitly (d features, image size H×W×C)
  2. Hidden layers: How many + neurons each + activation (ReLU default)
  3. Output layer: #neurons (K classes or 1) + activation (softmax/sigmoid/identity)
  4. Loss function: MSE / BCE / CCE — must match output activation
  5. Optimizer: Mini-batch SGD or Adam; specify batch size + η
  6. Justify everything with task-specific reasoning

Common pairings to memorise:

  • Regression: Identity output + MSE loss
  • Binary: Sigmoid output + BCE loss + threshold 0.5
  • Multi-class: Softmax output + CCE loss + argmax prediction
  • Hidden layers: ReLU (deep: Leaky ReLU)
🚨 Common Exam Mistakes to Avoid
  • Using Softmax in hidden layers — only for multi-class output
  • Using Sigmoid in output for multi-class — use Softmax (outputs don't sum to 1)
  • Forgetting bias in parameter count — always +nout per layer
  • Wrong CNN output formula — it's ⌊(n+2P−K)/S⌋+1, not (n+2P−K)/S
  • Forgetting pooling params = 0 — no learnable params in pooling
  • Saying accuracy is good for imbalanced data — use F1/Recall
  • Wrong gradient sign in code — it's (y_pred − y), not (y − y_pred) for GD
  • Using wrong axis in numpy softmax — always axis=1 for batch of examples
  • Claiming higher accuracy = better model — consider business cost and recall
